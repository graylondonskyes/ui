# Brand System

Default identity: kAIxU / Skyes Over London compatible premium UI.

Use existing project brand assets first.

If project has logos, colors, backgrounds, founder imagery, or brand CSS, preserve them.

Do not invent replacement brand assets.

Preferred effects:

- subtle glow
- floating logo/hero image treatment
- dark premium backgrounds
- gold/purple/neon accents where brand-compatible
- crisp cards
- readable panels
- strong CTA hierarchy

Never place dev notes or implementation commentary inside public/client-facing pages.
