# MCP Website Builder

## Goal

Use MCP to move from static guessing to live-doc and browser-inspected implementation.

✅ Context7 MCP for current docs.
✅ Playwright MCP for rendered UI inspection and click proof.
☐ If unavailable, report unavailable and continue with local build/smoke.

Workflow: repo intake, Context7 docs, local app start if possible, Playwright inspection, actual app-file upgrade, browser proof, build/test/smoke.

Build premium surfaces with strong hero, clear nav, polished sections, responsive grid, visible focus states, mobile-safe layout, functional CTAs, and no fake controls.
